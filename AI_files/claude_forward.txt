--------------------------------------------------------------------------------
1. WHAT THIS PROJECT IS
--------------------------------------------------------------------------------
jac313 = a versioned C++23 "umbrella" monorepo over a family of legacy libraries.
Each package re-exports a legacy API under jac313::<Name>::v001 while the original
implementation is migrated in small chunks (not a big-bang rewrite).

Packages (ALL now IN-TREE — no sibling repos needed to build):
  Qlite/  -> jac313::Qlite::v001  | IN-TREE header-only SQLite wrapper
                                    (migrated from jacQlite/jac::qlite; SQLite3 system dep)
  jText/  -> jac313::jText::v001  | IN-TREE compiled jtext_core; public API now in
                                    namespace jac313::jText::v001 (was global ::JTextWriter).
                                    Internal parser/validator stay in namespace jtext.
  store/  -> jac313::Store::v001  | IN-TREE core + persistence (ported from ts_store)

store is the real work: a time-series event store owning the hot path and persist
sinks (binary, jText, SQL, flag-routing). Matrix stress tests 001-008 (TS/XS +
flags) were ported from sibling ts_store.

Optional C++23 modules (import jac313.*.v001) exist; require Ninja + module scanning.

4. HOW TO BUILD  (C++23 required; modules need Ninja)
--------------------------------------------------------------------------------
g++-15 + clang are the STANDARD toolchains (the gate). g++-15 auto-selected when present.
g++-14 is an optional conservative fallback (--force-gcc14), NOT in the readiness gate.
clang++-20+ is a first-class peer (--clang); minimum Clang 20; modules need clang-scan-deps-20.

Manual CMake (pick compiler + build tree; separate trees avoid object mixing):
  cmake -G Ninja -S . -B build-gcc14 -DCMAKE_CXX_COMPILER=g++-14
  cmake -G Ninja -S . -B build-gcc15 -DCMAKE_CXX_COMPILER=g++-15
  cmake --build build-gcc15
  ctest --test-dir build-gcc15

Key CMake options (defaults):
  JAC313_BUILD_MODULES      OFF   C++23 module targets (Ninja only)
  JAC313_BUILD_STORE_TESTS  OFF   store unit + matrix tests in ctest
  JAC313_BUILD_TEST_CLI     ON    builds jac313_test_cli
  JAC313_BUILD_QLITE/JTEXT/STORE  ON
Build type DEFAULTS to Debug (dev, matrix gates). --release is now a FIRST-CLASS
profile: build_type is a recorded RunIdentity dimension, so Debug & Release results
stay distinct (committed baselines now include both). Release build dirs follow the
build-<label>-release convention (e.g. build-gcc15-release, build-clang20-release).

6. CONVENTIONS / GOTCHAS
--------------------------------------------------------------------------------
- No shell-script test harness by design — jac313_test_cli is the orchestrator.
- No GitHub Actions / CI yet — deliberate. Local feedback only.
- Namespace contract: everything under jac313::<Name>::v001. v001 reserves room
  for v002 without breaking pinned includes/imports. Legacy names still work.
- Debug is the DEFAULT; --release is first-class (build_type is a RunIdentity
  dimension — Debug/Release never conflate). Don't force a profile without asking.
- Commit/push ONLY when the user asks. Branches main/dev-work/test-work are all
  in sync at f51a1f9; user pushes to all three at wrap-up. (main is fast-forwardable.)
- Metrics are DB-only (file side-channels were killed). Don't reintroduce them.
- RESULTS PATH = RunIdentity: test-summary/<os>/<compiler>/<build_type>/<disk>/<size>/.
  os = sensed <id>-<version> (e.g. rhel-10.2; full version, 10.1 != 10.2). The identity
  is defined ONCE in run_identity.hpp; DB keys, the path, and rendered tables all DERIVE
  from it. Adding a new results dimension = a single deliberate edit there.
- matrix run is SELF-BUILDING (f51a1f9): if the target build dir has no built tests it
  configures+builds first (creating the dir), then runs. So `matrix run --clang --release
  --build-dir build-clang20-release` works on a fresh tree with NO manual `all` pre-step.
  --modules still takes the benchmarked module-build path.
- STALE-BINARY GOTCHA: after editing ANY tool source, rebuild the driver
    scl enable gcc-toolset-15 -- cmake --build build-gcc15-probe --target jac313_test_cli
  before running, or you run old logic (old schema/path) against new code. Bit us twice.
